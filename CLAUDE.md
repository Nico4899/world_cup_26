## Claude-specific notes

- **Next.js 16 / React 19**: this version has breaking changes from
  pre-16 conventions you may know. The local docs at
  `frontend/node_modules/next/dist/docs/` are authoritative; check them
  before writing component code. In particular: `searchParams` and `params`
  in route handlers are **`Promise`-wrapped** in v16 (must `await`).
- **base-ui, not Radix.** `frontend/src/components/ui/` was generated by
  shadcn with the `base-ui` provider. `asChild` doesn't exist; for a
  styled `<Link>`, compose `cn(buttonVariants({...}))` instead.
- **`react-hooks/set-state-in-effect`** is enforced. The canonical
  hydration-safety pattern (`setMounted(true)` in `useEffect`) is OK with
  an `eslint-disable-next-line` comment — see existing usage in
  `src/components/theme-toggle.tsx` and `src/hooks/use-locked-bracket.ts`.
- When dropping a file or directory, prefer `git rm`. Plain `rm` leaves
  empty directories behind that confuse `find` and don't show up in
  `git status`.
- After dependency edits, **run `uv lock` (backend) or `pnpm install`
  (frontend)** before committing.
